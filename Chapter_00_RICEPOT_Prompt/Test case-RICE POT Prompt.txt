E — EXAMPLE
════════════════════════════════════════════════════
PRD says:
"User must log in with 2FA. Role-based access 
controls must restrict features by user role."

POSITIVE test case expected:

LOGIN-TC001 | 
Verify that with valid credentials and correct 
OTP, login should be successful |
Pre-Condition: User account exists, 2FA enabled, 
browser is Chrome latest |
Steps:
  1. Navigate to VWO login page
  2. Enter valid registered email and password
  3. Click the Login button
  4. Enter valid OTP received on registered device
  5. Verify user is redirected to the dashboard |
Expected Result: User logs in successfully and 
lands on the role-appropriate dashboard |
Priority: P0 | Severity: Critical | 
Type: Positive | Misc: PRD Ref — Security, 2FA

NEGATIVE test case expected:

LOGIN-TC002 | 
Verify that with valid credentials and incorrect 
OTP, login should fail and show error message |
Pre-Condition: User account exists, 2FA enabled |
Steps:
  1. Navigate to VWO login page
  2. Enter valid email and password
  3. Click the Login button
  4. Enter an incorrect or expired OTP
  5. Verify error message is displayed |
Expected Result: Login fails. Error message 
"Invalid OTP, please try again" is displayed. 
User remains on the login page |
Priority: P0 | Severity: Critical | 
Type: Negative | Misc: PRD Ref — Security, 2FA

This depth is required for every module.

P — PARAMETERS
════════════════════════════════════════════════════
Input              : VWO PRD PDF (Product Requirements Document (PRD) attached)
TC ID Format       : [ModuleCode]-TC001 
                     e.g. LOGIN-TC001
                     HEATMAP-TC002
                     (unique across ALL modules —
                     never reset per module)
Module Grouping    : Group all TCs by module
Priority Levels    : P0 — Critical
                     P1 — High
                     P2 — Medium
                     P3 — Low
Severity Levels    : Critical / Major / 
                     Minor / Trivial
Test Types         : Positive, Negative,
                     Edge Case, Boundary,
                     UI Validation,
                     Integration / Flow
Minimum per Module : 2 Positive TCs
                     2 Negative TCs
                     1 Edge / Boundary TC
                     1 UI Validation TC
Pre-Conditions     : Required for every TC
                     (specific, not generic)
Test Steps         : Numbered, minimum 4 steps
PRD Reference      : Add in Misc/Comments column
                     as "PRD Ref — [Module], 
                     [Requirement Area]"
Ambiguous PRD      : If a requirement is unclear,
                     note it in Misc/Comments —
                     do not skip the test case
